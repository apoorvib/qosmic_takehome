# Eval Loop

The report is a rendered view; the artifact tree is the source of truth. Each run freezes `manifest.json`, page captures, screenshots, and `reason/*.json`. `validate.py` enforces hard gates: 10 experiments, all 5 pillars, resolvable evidence, store-specific bindings, confidence-band coherence, 3-4 competitors, and artifact-grounded tech checks. `score_report.py` adds deterministic rubric scores and emits `judge_tasks[]` for the few things scripts should not pretend to know: whether evidence supports the hypothesis, whether the recommendation is truly store-specific, and whether the executive summary faithfully reflects the top leaks.

The loop repairs at object-path granularity. A failure like `experiments[4].evidence.triggering_signal`, `experiments[7].evidence.store_profile_ref`, or `tech_checks[2].grounded_in` becomes a targeted prompt to regenerate only that JSON section, then re-run validation and scoring against the same frozen crawl. Judge failures are aggregated by type: unsupported evidence, generic recommendation, wrong priority, fabricated technical status, or missing major leak. Repeated failures do not stay as anecdotes; they become harness changes: schema tweaks, deterministic validators, reason-skill rubric updates, or new fixture cases. Every harness/skill edit must beat the previous version on the failing audit and a holdout set so the loop does not overfit one store.

Humans enter at calibration points, not every report. Early on, they label a thin sample of judge outputs (`agree`, `too harsh`, `too lenient`, `missed issue`) and business-level misses (`valid insight`, `unsupported`, `generic`, `wrong priority`, `missed major leak`). Those labels form a meta-eval: judge-vs-human agreement, false-pass rate on known-bad fixtures, and regression rate on holdouts. As agreement improves, the human surface shrinks from full audit review to low-confidence judge calls, then to periodic calibration batches and new failure-mode review.

In 1-3 months, the system should be self-improving in three lanes: deterministic checks absorb recurring mechanical failures, judge prompts/rubrics are tuned from labeled disagreements, and the reason/playbook skills evolve from clustered missed-leak patterns. Humans still own rubric changes and final calibration, but the day-to-day loop is autonomous: crawl, reason, write, validate, judge, repair, re-score, update the failure library, and promote only changes that improve held-out eval performance.
